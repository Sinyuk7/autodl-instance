# RAG 本地检索系统

基于 Git 增量索引的本地 RAG (Retrieval-Augmented Generation) 检索工具。

## 功能

- 🔍 **语义搜索** - 基于向量相似度检索相关文档
- 📦 **本地向量化** - 使用 sentence-transformers，无需 API
- 🚀 **Git 增量索引** - 只更新变化的文件，速度快
- 📋 **复制友好** - 输出可直接粘贴到 AI 对话

## 快速开始

### 1. 首次安装

```bash
# 运行安装脚本 (会自动创建 conda 环境)
python3 rag/rag_install.py
```

### 2. 使用

```bash
# 运行 RAG 工具
./rag/rag
```

### 3. 交互命令

```
❓ 你的问题: ComfyUI 如何安装?     # 搜索
❓ 你的问题: status                # 查看索引状态
❓ 你的问题: update                # 增量更新 (基于 Git)
❓ 你的问题: reindex               # 全量重建
❓ 你的问题: quit                  # 退出
```

## 文件说明

```
rag/
├── rag              # 启动脚本 (bash)
├── rag_indexer.py   # 核心索引器

├── rag_install.py   # 安装脚本
└── README.md        # 本文档
```

## 工作原理

```
┌─────────────────────────────────────────────────────┐
│ 1. 首次运行: 全量索引                                │
│    扫描仓库 → 切分文档 → 生成向量 → 保存索引         │
│    (约 1-2 分钟)                                    │
├─────────────────────────────────────────────────────┤
│ 2. 日常使用: 增量索引                                │
│    git diff → 找出变化文件 → 只更新这些文件          │
│    (几秒钟)                                         │
├─────────────────────────────────────────────────────┤
│ 3. 搜索: 语义匹配                                   │
│    问题 → 向量化 → 余弦相似度 → 返回 Top-K 结果      │
│    (毫秒级)                                         │
└─────────────────────────────────────────────────────┘
```

## 依赖

- Python 3.11 (通过 conda/miniforge)
- sentence-transformers
- numpy

## 索引存储

索引文件保存在项目根目录的 `.rag_index/`，已加入 `.gitignore`。

| 文件 | 说明 |
|------|------|
| `chunks.json` | 文档块文本 |
| `embeddings.npy` | 向量数据 |
| `config.json` | 配置 + Git commit |
